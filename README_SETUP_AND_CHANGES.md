# تِرياقي — دليل التشغيل وملخص التعديلات

هذا الملف منفصل عن `README.md` الخاص بالفريق. يوثّق **ربط الواجهات بالخادم (PHP + MySQL)**، **تطبيق Flutter**، و**طريقة التشغيل** على الجهاز.

---

## هيكل المشروع (بعد التعديلات)

| المسار | الوصف |
|--------|--------|
| `config/database.php` | إعدادات اتصال MySQL |
| `database_setup.sql` | إنشاء الجداول وقاعدة `smart_medicine` |
| `api/` | واجهات REST (PHP) |
| `index.html` + `script.js` + `api-config.js` | الواجهة الويب المربوطة بالـ API |
| `flutter_app/` | مشروع Flutter كامل (`lib/main.dart`، `lib/api/`) |

> ملاحظة: الملف `main.dart` في **جذر** المجلد `Teryaqi-main` قديم؛ التطبيق الفعلي موجود في **`flutter_app/lib/main.dart`**.

---

## المتطلبات

- **PHP** (مع PDO و MySQL)
- **MySQL** أو MariaDB
- **متصفح** حديث (للويب)
- **Flutter SDK** (لتطبيق الجوال فقط)

## 1) إعداد قاعدة البيانات

1. أنشئ قاعدة البيانات أو استورد السكربت:
   - من phpMyAdmin أو سطر الأوامر: تنفيذ محتوى `database_setup.sql`.
2. عدّل `config/database.php`:
   - `$host`، `$db_name` (مثلاً `smart_medicine`)، `$username`، `$password`.

---

## 2) تشغيل الخادم (Backend + الويب + تطبيق المحاكي)

من مجلد المشروع `Teryaqi-main` (اللي فيه مجلد `api`):

```bash
cd Teryaqi-main
php -S 0.0.0.0:8080
```

أو انقر `start-php-server.bat` من نفس المجلد.

ثم افتح في المتصفح على **الكمبيوتر**:

```text
http://localhost:8080/index.html
```

**ليش `0.0.0.0` وليس `localhost` فقط؟**  
- المتصفح على جهازك يتصل بـ `localhost` = نفس الجهاز → يكفي أن PHP يستمع على المنفذ.  
- **محاكي أندرويد** يتصل بجهازك بعنوان **`10.0.2.2`**؛ هذا يصل إلى «جهاز التطوير» فقط إذا كان الخادم يستمع على **كل الواجهات** (`0.0.0.0`). إذا شغّلت `php -S localhost:8080` فقط، أحياناً المحاكي لا يصل رغم أن الويب يعمل.

- الواجهة الويب تستخدم **`window.location.origin`** كعنوان API — لا إدخال يدوي.
- لا تفتح `index.html` عبر `file://`.

### هل Flutter يتصل بقاعدة البيانات مباشرة؟

**لا** — وهذا مقصود: التطبيق يتصل بـ **REST (PHP)** فقط؛ **PHP + PDO** هما اللذان يتصلان بـ **MySQL**. نفس منطق الويب؛ الفرق الوحيد **عنوان الشبكة** (محاكي ≠ متصفح على نفس الجهاز).

---

## 3) تشغيل تطبيق Flutter

من مجلد التطبيق:

```bash
cd Teryaqi-main/flutter_app
flutter pub get
flutter run
```

### عنوان API الافتراضي في التطبيق

| المنصة | العنوان الافتراضي (مثال) |
|--------|---------------------------|
| محاكي **Android** | `http://10.0.2.2:8080` (يصل إلى `localhost` على الكمبيوتر المضيف) |
| **iOS / Windows / macOS / Linux** | `http://127.0.0.1:8080` |
| **الويب (Chrome)** | `http://localhost:8080` |
| **هاتف حقيقي** على نفس شبكة Wi‑Fi | ضع IP جهاز الكمبيوتر، مثل `http://192.168.1.x:8080` |

للتجاوز اليدوي (جدار ناري، إلخ):  
`flutter run --dart-define=TERYAQI_API_BASE=http://192.168.x.x:8080`

### إعدادات الجهاز للتطوير

- **Android:** تم تفعيل `usesCleartextTraffic` في `AndroidManifest` لدعم HTTP أثناء التطوير.
- **iOS:** تمت إضافة `NSAllowsLocalNetworking` في `Info.plist` للاتصال بالخادم المحلي.

## 4) تدفق الاستخدام (ويب أو فلاتر)

1. **إنشاء حساب** (الحقول المطلوبة في الـ API تشمل الجنس `Male` / `Female` ورقم هوية فريد).
2. **تسجيل الدخول** — يُعاد `patient_id` ويُخزَّن محلياً (ويب: `localStorage`، فلاتر: `SharedPreferences`).
3. **إضافة دواء:** يتم بالتسلسل:
   - إنشاء سجل في جدول الأدوية الأساسية،
   - ربط الدواء بالمريض،
   - إضافة **جدولة** لوقت الجرعة.

## ملخص التعديلات البرمجية (ما تم تنفيذه)

### الواجهة الويب (`script.js`، `index.html`، `api-config.js`)

- ربط الواجهة بـ REST بدلاً من الاعتماد على `localStorage` فقط لبيانات الدواء.
- إصلاح النموذج: جمع الحقول داخل `<form>` واحد مع زر إرسال يعمل بشكل صحيح.
- عنوان الـ API يُشتق تلقائياً من **`window.location.origin`**؛ تسجيل الدخول والخروج و**حساب جديد** (نافذة حوار) في **الهيدر** دون حاوية منفصلة لعنوان الخادم.
- استدعاءات: تسجيل الدخول، التسجيل، جلب الأدوية، إنشاء دواء، ربط بالمريض، إضافة جدولة.

### الخادم PHP

- **`api/medications/get_for_patient.php`:** إرجاع قائمة فارغة `[]` بحالة HTTP 200 بدلاً من اعتبارها خطأ 404 عند عدم وجود أدوية؛ ومعالجة غياب `patient_id`.
- **`api/patients/register.php`:** إرجاع `patient_id` بعد التسجيل؛ التحقق من `gender`؛ دعم `OPTIONS` للـ CORS.
- **`includes/patient.php`:** حفظ `patient_id` بعد الإدراج الناجح.
- **`api/medications/add_schedule.php`:** (جديد) ربط دالة `createSchedule` في الكلاس بمسار HTTP.
- **`includes/medication.php`:** إضافة حقل `intake_time` في استعلام جلب أدوية المريض (من جدول الجدولة).
- **CORS:** تحديث عدة ملفات (`login`، `register`، `create_medications`، `add_for_patient`، `add_schedule`) لدعم طلبات ما قبل الإرسال `OPTIONS` عند الحاجة.

### تطبيق Flutter (`flutter_app/`)

- إنشاء مشروع Flutter قياسي داخل `flutter_app/`.
- إضافة الحزم: `http`، `shared_preferences`.
- ملف **`lib/api/teryaqi_api.dart`:** عميل API يطابق الويب.
- ملف **`lib/config/api_config.dart`:** عنوان افتراضي حسب المنصة.
- **`lib/main.dart`:** لوحة التحكم مع بطاقة المصادقة، الجلسة، وجلب/إضافة الأدوية عبر الشبكة.
- تعديلات **Android / iOS** لدعم الاتصال HTTP بالخادم المحلي أثناء التطوير.

## استكشاف الأخطاء السريع

| المشكلة | اقتراح |
|---------|--------|
| لا يتصل التطبيق بالخادم | تأكد أن `php -S` يعمل، والعنوان في التطبيق يطابق IP/المنفذ الصحيح. |
| Android حقيقي لا يصل | استخدم IP الشبكة المحلية للكمبيوتر وليس `localhost`. |
| خطأ قاعدة البيانات | راجع `config/database.php` وتشغيل `database_setup.sql`. |
| تسجيل فاشل | رقم الهوية والبريد يجب أن يكونا فريدين؛ الجنس `Male` أو `Female`. |
